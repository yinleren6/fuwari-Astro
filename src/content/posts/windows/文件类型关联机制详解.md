## 引言

双击一个 `.txt` 文件，记事本打开了。这背后是 Windows 文件关联机制在起作用。理解这个机制，你就能自定义任何文件后缀的打开方式，甚至创建自己的文件类型。

## 关联机制的两步走

Windows 的文件关联分两步：

```
双击 .txt 文件
    ↓
assoc .txt  → 查到 .txt=txtfile    ← 第一步：后缀 → 文件类型
    ↓
ftype txtfile → 查到打开命令        ← 第二步：文件类型 → 程序
    ↓
用记事本打开
```

### 命令行工具

| 命令 | 作用 |
|------|------|
| `assoc` | 显示所有后缀关联 |
| `assoc .txt` | 查询 .txt 对应的文件类型 |
| `ftype` | 显示所有文件类型的打开命令 |
| `ftype txtfile` | 查询 txtfile 类型的关联程序 |

### 实际演示

```batch
assoc .txt     → .txt=txtfile
assoc .doc     → .doc=Word.Document.8
assoc .exe     → .exe=exefile
ftype exefile  → exefile="%1" %*
```

修改关联也简单：

```batch
;将 .txt 关联到 Word
assoc .txt=Word.Document.8

;恢复为记事本
assoc .txt=txtfile
```

> **注意：** `assoc` 和 `ftype` 是 cmd 的内部命令，PowerShell 需要 `cmd /c assoc` 才能使用。

## 注册表方式：创建自定义文件类型

命令行虽然方便，但注册表方式更灵活——可以自定义图标、右键菜单文字、文件类型描述等。

### 创建一个 .aaa 文件类型（用记事本打开）

```reg
Windows Registry Editor Version 5.00

;告诉系统 .aaa 后缀对应 aaa_test_file 类型
[HKEY_CLASSES_ROOT\.aaa]
@="aaa_test_file"

;定义打开命令
[HKEY_CLASSES_ROOT\aaa_test_file\shell\open\command]
@="\"C:\\Windows\\System32\\notepad.exe\" \"%1\""

;（可选）文件类型描述
[HKEY_CLASSES_ROOT\aaa_test_file]
@="假的文件"

;（可选）自定义图标
[HKEY_CLASSES_ROOT\aaa_test_file\DefaultIcon]
@=""

;（可选）自定义右键默认文字
[HKEY_CLASSES_ROOT\aaa_test_file\shell\open]
@="让我瞅瞅(&O)"
```

导入后，双击 `.aaa` 文件就会用记事本打开，右键菜单显示"让我瞅瞅"。

### 清除关联

```reg
Windows Registry Editor Version 5.00

[-HKEY_CLASSES_ROOT\.aaa]
[-HKEY_CLASSES_ROOT\aaa_test_file]
```

## 修改 exe 文件的"打开"文字

如果你想搞怪，可以把 exe 文件的右键"打开"改成"运行"：

```reg
[HKEY_CLASSES_ROOT\exefile\shell\open]
@="运行(&O)"
```

## 小结

Windows 文件关联的机制其实很清晰：`assoc` 连接后缀和类型，`ftype` 连接类型和程序。如果需要更高级的自定义（图标、描述、右键菜单），直接操作注册表 `HKCR` 即可。

> 动手试一下：创建一个 `.笔记` 后缀，用 VS Code 打开——日常工作中很实用。
